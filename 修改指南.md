## 一、HTML 头部信息 (index.html 第 1-42 行)

### 1.1 Meta 标签信息
**位置**: `index.html` 第 5-9 行


## 五、结果展示轮播 (index.html 第 195-250 行)

### 5.1 视频项
**位置**: `index.html` 第 199-246 行

每个视频项的格式：
```html
<div class="item item-名称">
  <video poster="" id="名称" autoplay controls muted loop playsinline height="100%">
    <source src="./static/videos/视频文件名.mp4" type="video/mp4">
  </video>
</div>
```

**修改说明**:
- 根据您的实际视频数量添加或删除 `<div class="item">` 块
- 修改 `item-名称`、`id="名称"` 和视频文件路径
- 如果视频较少，可以删除不需要的项

**示例**（假设您有 3 个结果视频）:
```html
<div class="item item-result1">
  <video poster="" id="result1" autoplay controls muted loop playsinline height="100%">
    <source src="./static/videos/result1.mp4" type="video/mp4">
  </video>
</div>
<div class="item item-result2">
  <video poster="" id="result2" autoplay controls muted loop playsinline height="100%">
    <source src="./static/videos/result2.mp4" type="video/mp4">
  </video>
</div>
<div class="item item-result3">
  <video poster="" id="result3" autoplay controls muted loop playsinline height="100%">
```

## 💡 提示

1. **备份原文件**: 修改前建议先备份 `index.html`
2. **逐步修改**: 按照优先级逐步修改，每完成一部分可以预览效果
3. **测试链接**: 确保所有外部链接都能正常访问
4. **视频格式**: 建议使用 MP4 格式，确保浏览器兼容性
5. **文件大小**: 注意视频和图片文件大小，过大可能影响加载速度
6. **路径检查**: 确保所有相对路径（`./static/...`）正确指向文件

## 🔧 常见问题

**Q: 如果我没有某些内容（如代码、数据集）怎么办？**
A: 直接删除对应的链接按钮块即可。

**Q: 插值功能是必须的吗？**
A: 不是。如果您的论文没有这个功能，可以删除整个动画插值部分。

**Q: 如何测试修改后的网站？**
A: 可以在本地用浏览器直接打开 `index.html` 文件，或使用简单的 HTTP 服务器（如 Python 的 `python -m http.server`）。

**Q: 如何部署到 GitHub Pages？**
A: 将修改后的文件推送到 GitHub 仓库，在仓库设置中启用 GitHub Pages，选择主分支即可。
